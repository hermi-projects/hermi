# Hermi: AI 原生软件工厂 (AI-Native Software Factory)

Hermi 不仅仅是一个 Java 框架，它是为 AI 时代量身定制的**工程标准**。通过强制执行**蓝图优先编排 (Blueprint-First Orchestration)**，Hermi 将传统的软件开发重塑为一个可预测、可扩展且自治的 **AI 软件工厂**。

## 1. 核心层级：宏观愿景 —— 从“手工业”到“自动化流水线”

*   **核心理念**：将开发从“混合了逻辑决策与重复搬砖”的重体力劳动，拆解为**“纯净意图发现”**与**“自动化技术交付”**。
*   **目标**：实现 **One Human, Many Agents**。让一名架构师通过指挥 AI 集群，以工业级精度和并发速度交付系统。

---

## 2. 顶层策略：战略意图扫描 (Strategic Intent Sweep)

为了适配 AI 的上下文限制与逻辑特性，我们将传统的“自顶向下、广度优先”升级为**战略意图扫描**：
*   **横向全景 (Horizontal)**：在 Phase 1 必须完整扫清业务流程的所有路径。
*   **纵向隔离 (Vertical)**：在业务逻辑未完全闭环前，严禁向下钻探任何技术细节（数据库、API 等）。
*   **价值**：确保 AI 的工作始终围绕“经过验证的蓝图”展开，消除无效返工。

---

## 3. 生命周期：发现与实现的分离

### Phase 1：蓝图编排 (Blueprint-First Orchestration)
*   **协作方式**：人机深度对话（Human Architect + Architect Agent）。
*   **核心方法**：**叙事优先发现 (Narrative-First Discovery)**。在编写业务叙事的过程中，代码即时发现所需的 I/O 契约。
*   **终点**：通过 **Test Shell Gate**，逻辑在内存中 100% 跑通并被证明正确。

### Phase 2：战术交付 (Tactical Implementation)
*   **执行者**：**Specialized Agent Pool**（领域专家 Agent 集群）。
*   **特性**：物理隔离、高度并行的自动化交付。

---

## 4. 协作协议：工单 (Work Order)

这是连接“蓝图”与“零件”的数字接口，也是防止 AI 幻觉的核心：
*   **约束扩散 (Constraint Diffusion)**：将业务层的校验注解自动扩散为数据库 DDL 或 API 校验规则。
*   **文档感知 (Doc-Aware)**：Dispatcher 自动为工单匹配所需的 OpenAPI 或 Schema 文档。
*   **语义错误映射 (Semantic Error Mapping)**：强制 AI 在底层技术异常与业务异常之间建立转换映射。

---

## 5. 微观执行：双阶段验证 (Dual-Phase Verification)

确保 AI 产出的每一行代码都符合工业标准：
1.  **Worker Phase (施工)**：专项 Agent 基于工单完成适配器逻辑编写。
2.  **Review & QA Phase (审计)**：
    *   **Review Agent**：检查是否存在“技术泄露”（严禁在适配器中写业务逻辑）。
    *   **Test-Shell Agent**：自动生成并在隔离容器中运行集成测试，证明适配器真实可用。

---

## 6. 工程优势：上下文隔离 (Context Isolation)

Hermi AI 工厂的“超能力”在于其极致的模块化：

| 能力 | 对 AI 协作的影响 |
| :--- | :--- |
| **沙箱任务 (Sandbox)** | 每个 Agent 在 <2k Token 的上下文内工作，准确率极高。 |
| **单向依赖 (One-Way)** | 物理防止了底层的“幻觉”污染业务核心。 |
| **即时契约发现 (JIT)** | Prompt 在需求产生时精准生成，匹配开发心流。 |
| **确定性校验** | 高层逻辑在实现前已被证明正确，缩小了错误的影响范围。 |

> [!IMPORTANT]
> **总结**：通过将“写代码”转化为“蓝图验证”与“零件组装”，Hermi 让一名人类架构师能够以高科技生产线的精度，领导由 AI Agent 构成的开发军团。
